# Execution Plan: Single-Window Claude→Gemini Auto-Execute Flow with Rate Limit Fix

Created: 2026-04-01 12:00:00
Request: 하나의 창에서 명령만 내리면 Claude가 계획을 작성하고 Gemini가 자동으로 구현하는 flow 구축 + rate limit 문제 해결
Status: PENDING

## Goal
Claude Code 세션 안에서 plan 작성 후 자동으로 Gemini 실행이 트리거되고, rate limit 발생 시 exponential backoff + Gemini REST API 직접 호출 fallback까지 갖춘 완전 자동화 flow 구현

## Current State
- `bin/cplan` 스크립트가 `gemini -p "..."` 로 programmatic 실행 → rate limit 에러 반복
- 별도 터미널에서 `gemini` interactive 모드 실행 시 정상 동작 (OAuth 인증 = 더 높은 quota)
- `commands/execute-gemini.md` 슬래시 커맨드로 수동 트리거 가능하나 자동화 아님
- `profiles/plan-only/CLAUDE.md` 에 plan-only 제약 정의됨

## Root Cause Analysis
- `gemini -p` (one-shot prompt) 모드는 API key 기반 인증을 사용할 가능성 높음 → free tier 제한 (분당 토큰 제한이 낮음)
- Interactive 모드는 OAuth 인증 → Google AI Studio 무료 tier 와는 별도 quota
- fallback 모델 루프가 retry 간격 없이 즉시 다음 모델 시도 → burst 요청으로 rate limit 가중

## Execution Plan

### Phase 1: Rate Limit 근본 해결 — cplan 스크립트 개선
Goal: `bin/cplan`의 `run_gemini_execute()` 함수를 개선하여 rate limit 문제를 해결

Tasks:
- [ ] `bin/cplan`의 `run_gemini_execute()` 함수에 **exponential backoff** 로직 추가
  - 첫 번째 실패: 30초 대기, 두 번째: 60초, 세 번째: 120초
  - 대기 중 사용자에게 countdown 표시 (`log_info "⏳ Rate limit — 30초 후 재시도 (gemini-2.5-flash)..."`)
- [ ] 동일 모델에 대한 재시도 로직 추가 (현재는 다른 모델로만 fallback)
  - 각 모델당 최대 2회 재시도 (backoff 포함)
  - 전체 시도 순서: model1(시도1) → model1(시도2, +30s) → model2(시도1, +60s) → model2(시도2, +90s) → model3...
- [ ] `GEMINI_FALLBACK_MODELS` 배열에 `gemini-2.0-flash` 추가 (가장 낮은 우선순위, 가장 안정적)

Completion criteria: `cplan -g <plan>` 실행 시 rate limit 에러 발생 → 30초 대기 후 재시도하는 로그 확인

### Phase 2: Gemini REST API 직접 호출 fallback 추가
Goal: gemini CLI가 모두 실패할 경우, Gemini REST API를 curl로 직접 호출하여 plan을 실행하는 fallback 경로 추가

Tasks:
- [ ] `bin/cplan`에 `run_gemini_api_fallback()` 함수 추가
  - `GEMINI_API_KEY` 환경변수 확인 (없으면 skip)
  - `curl`로 `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent` 호출
  - plan 파일 내용을 읽어서 "이 plan을 실행할 shell 명령어 스크립트를 생성해줘" 프롬프트로 전달
  - 응답에서 shell 스크립트 추출 → `/tmp/cplan-api-exec.sh`로 저장 → 사용자 확인 후 실행
- [ ] `run_gemini_execute()` 끝에서 모든 CLI 모델 실패 시 `run_gemini_api_fallback()` 호출
- [ ] API 호출에도 exponential backoff 적용 (429 응답 시)

Completion criteria:
- `GEMINI_API_KEY` 가 설정된 상태에서 CLI 모든 모델 실패 → API fallback으로 shell 스크립트 생성 확인
- 생성된 스크립트가 `/tmp/cplan-api-exec.sh`에 저장되고 내용이 plan의 task와 일치

### Phase 3: Claude Code Hook으로 자동 트리거 설정
Goal: Claude Code 세션에서 plan 파일이 Write될 때 자동으로 Gemini 실행을 트리거하는 hook 구성

Tasks:
- [ ] `.claude/settings.json`에 PostToolUse hook 추가:
  ```json
  {
    "hooks": {
      "PostToolUse": [
        {
          "matcher": "Write",
          "hooks": [
            {
              "type": "command",
              "command": "bash -c 'if [[ \"$TOOL_INPUT\" == *docs/superpowers/plans/*.md* ]]; then echo \"PLAN_CREATED: auto-triggering gemini execution\"; nohup cplan -g > /tmp/cplan-auto-exec.log 2>&1 & echo \"PID=$!\"; fi'"
            }
          ]
        }
      ]
    }
  }
  ```
- [ ] hook이 plan 파일 Write 시에만 트리거되도록 경로 패턴 검증 로직 확인
- [ ] `/tmp/cplan-auto-exec.log` 로그 파일 위치를 사용자에게 안내하는 메시지 출력

Completion criteria: Claude Code 세션에서 `docs/superpowers/plans/` 에 파일을 Write하면 자동으로 `cplan -g`가 백그라운드에서 시작됨
